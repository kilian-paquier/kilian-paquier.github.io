Flotte de camions

Lors de mon année de mobilité à l'Université du Québec à Chicoutimi, j'ai pu travailler sur un projet de développement en .NET (application web MVC avec Razor et EF6 - base de données MS SQL -). Par équipe, nous devions réaliser une application dite \"d'entreprise\" afin de manipuler le développement de grosses applications (de nombreuses classes - au minimum uen vingtaine - avec de nombreuses fonctionnalités incluant des modèles création / modification / visualisation / suppression des entités). Dans mon cas, cette application portait sur la gestion d'une flotte de camions. Nous avons donc (mon équipe) organisé l'application sous forme de commandes pour les clients. Les clients peuvent s'enregistrer et se connecter puis passer une commande dans laquelle ils vont fournir les marchandises qu'ils veulent transporter, la date de départ souhaitée ainsi que le lieu de départ et le lieu d'arrivée. Par la suite le client peut consulter ses commandes et les livraisons associées. Côté \"opérateur\", on peut gérer les camions, remorques, les employés et les commandes. C'est-à-dire qu'une fois la commande réalisée par le client celle-ci sera associée à une ou plusieurs livraisons par l'opérateur qui y ajoutera le camion de transport, le conducteur, la remorque, la date de départ réelle et la date d'arrivée estimée.
